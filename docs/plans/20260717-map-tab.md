# Вкладка «Карта»: трек + взятые КП на оффлайн-подложке (MBTiles)

## Overview

Четвёртая вкладка «Карта»: живой GPS-трек команды и взятые КП поверх растровой
оффлайн-подложки. Подложка — один MBTiles-файл на гонку, который сервер нарезает
заранее из существующих XYZ-тайлов и отдаёт статикой; пользователь скачивает его
явной кнопкой до старта (по Wi-Fi) — дальше карта работает полностью оффлайн.
Без скачанного файла вкладка деградирует в Apple-подложку (онлайн) с тем же
треком и КП.

Ключевые решения (из брейншторма):
- Координаты КП **не** приходят с сервера — берём GPS-фикс самой отметки
  (`Mark.locLat`/`locLon`, записывается `attachLocation` при взятии). Отметки
  без фикса на карте не показываются — это осознанно.
- Оффлайн-доставка — **один файл**, не покрайловый обход тайлов: атомарно,
  размер известен, докачка = перезапуск с нуля (resume data не используем).
- Рендер — MapKit (`MKMapView` + сабкласс `MKTileOverlay`), **без** новых
  зависимостей: GRDB уже в проекте и читает sqlite-файл MBTiles.

## Technical Details

**Формат MBTiles** (стандарт): таблица `tiles(zoom_level, tile_column, tile_row,
tile_data)` + таблица `metadata(name, value)` с ключами `bounds` («W,S,E,N»),
`center`, `minzoom`, `maxzoom`, `format`. Главная ловушка — **строки в
TMS-схеме**: `tile_row = 2^z − 1 − y` (y — XYZ/«гугловский»). Флип живёт в одном
месте — `Core/Map`.

**Хранение файла**: `Application Support/maps/<raceId>.mbtiles` (рядом с
`kolco24.db`, прецедент `PhotoStorage`). Наличие файла = «карта скачана»,
отдельного флага в prefs нет. Скачивание: временный файл → атомарный
`FileManager.moveItem` (файл появляется только целым); обрыв/отмена → temp
удаляется, состояние `notDownloaded`.

**Машина состояний `MapModel.MapAvailability`**:
`noMapForRace` (у гонки `mapUrl == nil`) / `notDownloaded` / `downloading(progress: Double)`
/ `ready(path: String)` / `failed(message: String)` (→ тост + возврат в
`notDownloaded`).

**Данные для рендера** (всё готовит модель, `TrackMapView` получает значения):
- трек: `trackUsable` (`sortedTrackPoints(filterPoints(points))` — как в
  `TeamModel`), живое обновление полной заменой `MKPolyline`;
- КП: отметки с `locLat != nil` → `MapMarkPin { lat, lon, number, cost, timeMs }`;
  номер — через `checkpointsById[mark.checkpointId]`, стоимость — live-цена с
  фолбэком `checkpointCosts[id] ?? mark.cost` (идиома `MarksModel.costOf`),
  время — `trustedTakenAt ?? takenAt`;
- камера: стартовый регион из `bounds` метаданных, без файла — по треку/КП;
  при активном оверлее `cameraBoundary` по bbox + `cameraZoomRange` из зумов.

**Рендер**: оверлей добавляется только при `ready` — `canReplaceMapContent =
true` (Apple-тайлы не грузятся вообще), `tileSize` 256, `minimumZ`/`maximumZ` из
метаданных. Без файла оверлея нет → штатная Apple-подложка (онлайн-fallback
бесплатно). Маркер КП — кружок `brandRed` с белым номером (`Font.mono`),
коллаут «КП 32 · 4 балла · 14:07». `showsUserLocation = true` (When-In-Use уже
запрашивается сканером; нет разрешения — точка просто не появится).

**Серверный контракт** (вне плана, делает владелец бэкенда): `GET /app/races/`
отдаёт опциональный `map_url` — прямой URL статики `.mbtiles` (без
HMAC-подписи: тайлы не секрет, download-task подписывать незачем).
**`map_url` обязан быть HTTPS** — ATS в `Info.plist` разрешает cleartext только
для локальной сети; HTTP с обычного хоста будет заблокирован системой. Ориентир
размера: район 20×20 км до z15 ≈ 1000–1500 тайлов, ~20–60 МБ.

**Типы координат**: `MapModel` (и `MapMarkPin`) оперируют только парами
`Double` lat/lon — `CLLocationCoordinate2D` появляется исключительно внутри
`Map/TrackMapView` (иначе `App/MapModel` потребовал бы `import CoreLocation`,
ломая grep-инвариант).

## Post-Completion

**Серверная часть** (владелец бэкенда, вне репозитория):
- поле `map_url` в `GET /app/races/` (null = карты нет);
- генерация `<race>.mbtiles` из XYZ-тайлов (bbox района + зумы ~z8–15,
  `mb-util`/`gdal2tiles`/скрипт) и раздача статикой без подписи.

**Ручная проверка на устройстве:**
- скачать карту по Wi-Fi → авиарежим → вкладка «Карта»: подложка, трек и КП
  рендерятся, камера ограничена районом;
- обрыв сети посреди скачивания → «ошибка» тостом, повторная попытка с нуля;
- гонка без `map_url` → Apple-подложка онлайн, строка о недоступности;
- удаление карты в настройках → вкладка возвращается в `notDownloaded`.
